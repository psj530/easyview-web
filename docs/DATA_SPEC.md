# 데이터 규격 (DATA SPEC)

> 원본 CSV 파일의 컬럼 정의 및 규격
> 새로운 고객사 데이터 적용 시 이 규격을 준수해야 합니다.

---

## 개요

| 항목 | 내용 |
|------|------|
| 파일 위치 | `backend/input/` |
| 인코딩 | `cp949` (한글 Windows) 또는 `UTF-8` (자동 감지) |
| 구분자 | 쉼표 (`,`) |
| 파일 형식 | CSV |

| 5 | 관리계정 | O | 관리 기준 분류 | 현금및현금성자산 |
| 6 | **구분** | O | BS 고정값 | BS |

## 2. JE (전표 - Journal Entry)

### 파일명
`JE.csv` (또는 파일명에 "JE"가 포함된 CSV)

### 컬럼 정의 (18개)

| 순서 | 컬럼명 | 필수 | 설명 | 예시 |
|------|--------|------|------|------|
| 1 | **일자** | O | 전표일자 (YYYY-MM-DD) | 2024-01-01 |
| 2 | **전표식별번호** | O | 전표 고유번호 | 20240101-0001 |
| 3 | **차대** | O | 차변/대변 구분 | 차변 |
| 4 | **금액** | O | 금액 (숫자) | 87732480 |
| 5 | 거래처 | O | 거래처명 | AA면세점 |
| 6 | 거래처(번역) | - | 거래처 번역명 | AA면세점 |
| 7 | 적요 | - | 적요 내용 | 1월분 임대료 |
| 8 | 적요(번역) | - | 적요 번역 | 1월분 임대료 |
| 9 | 계정코드 | O | 계정 코드 | 2130102 |
| 10 | 회사계정 | O | 회사 자체 계정명 | 미지급금(거래처원화) |
| 11 | 1차 번역 | - | 계정 번역명 | 미지급금(거래처원화) |
| 12 | **계정과목** | O | 계정과목명 | 미지급금(거래처원화) |
| 13 | **관리계정** | O | 관리 기준 분류 | 미지급금 |
| 14 | **공시용계정** | O | 공시 기준 분류 | 기타유동금융부채 |
| 15 | 합산계정 | O | 유동부채 등 | 유동부채 |
| 16 | **분류** | O | 비용/자산/부채/수익/손익대체 | 부채 |
| 17 | **구분** | O | BS/PL/IT | BS |
| 18 | RecordID | O | 레코드 고유번호 | 4 |

### 데이터 규칙
- **일자**: `YYYY-MM-DD` 형식 필수
- **차대**: `차변` 또는 `대변`
- **구분**: `BS` (재무상태), `PL` (손익), `IT` (내부거래)
- **분류 (PL)**: `수익`, `비용`, `손익대체`
- **분류 (BS)**: `자산`, `부채`
- **관리계정**: SGA 세부 항목 집계에 사용 (급여, 광고선전비, 경상개발연구비 등)

## 3. 새 고객사 데이터 적용 절차

1. TB, JE CSV 파일을 위 규격에 맞게 준비
2. `backend/input/` 폴더에 `TB.csv`, `JE.csv`로 저장
3. 백엔드 재시작 (자동으로 데이터 로드)
4. 또는 `/api/upload` 엔드포인트로 업로드 (재시작 불필요)

### 주의사항
- 컬럼 순서가 반드시 위 정의와 동일해야 합니다
- 인코딩은 `cp949` 또는 `UTF-8` (BOM 포함/미포함 모두 지원)
- 금액 컬럼의 쉼표, 공백, 괄호(음수) 모두 자동 처리됩니다
- 당기/전기 구분은 JE의 일자 기준으로 자동 처리됩니다 (2025=당기, 2024=전기)
